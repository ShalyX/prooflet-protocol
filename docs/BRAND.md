# Prooflet Brand Kit

## Product Definition

**Product name:** Prooflet
**Tagline:** Tiny agent jobs. Verified by proof. Paid in USDC.
**Short description:** Prooflet turns tiny AI-agent tasks into verified, payable work.
**Expanded description:** Prooflet lets issuers fund micro-jobs, lets autonomous agents complete them, verifies objective proof packets, routes subjective work through a GenLayer adjudication path, and settles approved work with Arc Testnet USDC through a treasury-controlled process.

## Visual Direction
Prooflet feels like a proof ledger and settlement desk for autonomous agent work.
**Visual metaphor:** micro-task → proof packet → verification stamp → payable state → USDC settlement receipt

*Note: The product name remains Prooflet. "Proof Desk" is only the visual design language, not the brand name.*

**Avoid:**
* Generic robot/AI logos
* Generic blockchain cubes
* Neon hacker dashboards
* DeFi token logos
* "Proof Desk" as the product name

## Brand Colors

* **Ink background:** `#15130E`
* **Surface:** `#1E1A16`
* **Border:** `#3A3024`
* **Amber:** `#F2A93B`
* **Soft amber:** `#FFD98A`
* **Text:** `#F5EFE6`

## Typography
Use clean, ledger-style fonts suitable for dashboards, data tables, and verification receipts. Sans-serif fonts like Inter, Roboto, or system UI fonts are preferred for clarity.

## Copywriting Guidelines

### Words to use:
* proof packets
* payable proofs
* treasury-controlled settlement
* Arc Testnet USDC
* GenLayer adjudication path
* dry-run sends nothing
* approved work only

### Words to avoid:
* instant settlement
* auto-pay from hosted API
* mainnet
* escrow contract, unless implemented
* cryptographically secure proofs, unless specifically proven
* live GenLayer finalized, unless actual live finalization happened
* Proof Desk as product name

## Screenshot Guidance
When presenting Prooflet in screenshots or documentation:
* Do not introduce fake metrics or fake transaction hashes.
* Highlight the "proof packet → verification → Arc Testnet USDC settlement" flow.
* Ensure data tables appear compact and professional, without UI distortion.
* When showing settlement, ensure it explicitly mentions "Arc Testnet USDC" and "dry-run" modes where applicable.
